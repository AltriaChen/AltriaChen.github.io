安防产品概述

发表于 2023-04-16 更新于 2023-11-04 分类于 科研

本篇是安防产品概述，介绍了 IPC，NVR，Switch。

一. IPC 概述

网络摄像机 (IPC，IP Camera) 是一种结合传统摄像机与网络技术所产生的新一代摄像机，可以将影像通过互联网传输到任何有网络覆盖的地方，网络上的用户可以直接使用浏览器观看 web 服务器上的影像。

(A) IPC 硬件架构

(a) 硬件架构

IPC 硬件上主要由镜头 (lens)、光电成像器件 (sensor)、视频编码处理器 (如 DSP)、主控芯片、照明模块、存储设备、音频接口、通信接口等部分组成。

IPC 剖面图如下

(b) Sensor

图像传感器 (Image Sensor)，或称感光元件，是一种将光学图像转换成电子信号的设备。早期采用光导摄像管，现在主要采用 CCD 和 CMOS。

  • Sensor 相关概念

    1. 靶面尺寸：用于表征 Sensor 感光面积的大小，一般去 Sensor 对角线的尺寸

    2. 最低照度：照度是指被光源照亮的物体，其单位面积上通过的光通量。通俗地说，就是物体被照亮的程度，单位是 Lux。
      最低照度表征摄像机灵敏度，是描述摄像机能在多黑的条件下看到可用的影像的指标。

      – 普通型：正常工作所需照度为 1~3 Lux。

      – 月光型：正常工作所需照度为 0.1 Lux 左右。

      – 星光型：正常工作所需照度为 0.01 Lux 以下。

      – 黑光型：正常工作所需照度为 0.001 Lux 以下。

      – 红外/白光照明型：零照度，采用红外/白光光源成像。

(c) 镜头

镜片：全玻璃或玻塑混合。玻塑混合镜头成为安防镜头的趋势。

  • 焦距

    焦距是光学系统中衡量光的聚集或发散的度量方式，指从透镜中心到焦点的距离。

  • 光圈

    光圈是在摄像机上用来控制镜头通光孔径大小的部件。

相对孔径：光圈直径 D 与焦距 f 的比值，镜头的相对孔径决定成像的照度。

F 值：通常以 F 值标示光圈大小，为相对孔径的倒数。F = f/D，F值越小，光圈越大。

(B) 成像

(a) 主光线角度

主光线就是从物点出射，通过孔径光阑的中心最后到达像点的光线。主光线角度 (Chief Ray Angle) 为主光线与平行光线的角度。

若镜头 CRA < Sensor CRA，光线达不到 pixed 边缘，四周偏暗。

若镜头 CRA > Sensor CRA，光线经 Micro lens 后会到达邻近的 pixed，导致 pixed 之间出现串扰、图像偏色。

(b) 景深

景深是指相机对焦点前后相对清晰的成像范围

(c) 畸变

畸变 (Distortion) 指光学系统对物体所成的像相对于物体本身而言的失真程度。畸变并不影响像的清晰程度，但影响像和物的几何相似性。

(C) 夜视模块

(a) IR LED

红外 LED：红外光属于非可见光，经物体反射后进入镜头进行成像。摄像机的 CCD/CMOS 图像传感器能够感受可见光、红外线、紫外线等，所以在没有可见光的情况下，红外摄像机可以拍摄到肉眼看不到的影像。

IR LED 属于电流型器件，其发光亮度、功率和流过的电流有关。IR LED 的驱动电路是一个恒流电路。

IR LED：

插件式：

  1. 单颗功率小，通常电流不超过 100mA
  2. 光衰快，环氧树脂材料受热容易老化断裂
  3. 散热差，热电不分离，均通过两个管脚散发

点阵式：

  1. 功率高，是插件式 LED 的十几倍
  2. 使用硅胶作为封装材料，寿命长
  3. 贴片式封装，散热佳

(b) IR-CUT

IR-CUT 双滤镜是指在摄像头镜头组里内置了一组滤镜，当镜头外的红外感应点侦测到光线的强弱变化后，内置的IR-CUT 自动切换滤镜能够根据外部光线的强弱随之自动切换，使图像达到最佳效果。也就是说，在白天或黑夜下，双滤光片能够自动切换滤镜，因此不论是在白天还是黑夜下，都能得到最佳成像效果。

可见光–用红片，滤除红外光，消除偏色；红外光–用白片，所有光线全透，黑白图像

(D) 无线配网

IPC 无线配网方式主要有两种：softAP，SmartConfig

(a) softAP

  • 方案简介

AP 与 Station可以同时工作，手机 APP 通过 AP 连接告知 IPC 路由的配置信息，而 Station 则去连接路由。当连接成功后，IPC 关闭 AP，保存路由信息进入 Station 模式持续工作。后续 IPC 开机 WiFi 将处于 Station 模式，并自动去连接已保存的路由。

  • 流程原理

用户使用手机连接 IPC 的 AP，通过 AP 与 IPC 交互，控制 IPC 的 Station 去关联路由，连接结果返回给手机端，当 IPC 接入路由器无线网络成功后，手机也接入路由，形成局域网即可添加 IPC 进行预览。

实现要求：WiFi 芯片需支持 AP+STA 共存。
特点：通用性强，与任何路由都可以配合使用，IPC 连接路由过程中不切换模式，可以保持与手机连接，给 APP 传递连接路由的阶段和状态。
局限性：操作较为复杂，用户体验不好，单 WiFi 芯片可能对 AP+Station 支持不够完美。

(b) SmartConfig

  • 方案简介

SmartConfig 简单来说就是 IPC 设备通过抓取空气中手机 (或路由转发) 出的特定数据包进行分析，得到 AP 的信息进行连接。
SmartConfig 是由 TI 提出的，并在 CC3000 上首次使用。后续由该技术引申出许多类似的技术，如 Realtek 的 SimpleConfig、Marvell 的 EasyConnect、微信的 AirKiss 等。
在链路未建立且数据是加密的情况下，设备拿到 Wi-Fi 帧，除了 MAC 帧头及数据长度能识别外，其他数据内容无法识别。

  • 基本原理

WiFi 帧的数据内容经加密处理，未关联不能识别，但是其长度可识别。IPC 根据抓到的 WiFi 数据帧长度进行解析，提取 AP 的接入信息，然后进行连接。
我们实现方案的抓包数据长度限制在 300bit 以上，数据帧加密密文与原文长度呈线性关系 y=x+C 。
AES、TKIP、CCMP 等加密算法密文与原文长度关系

  • 流程原理

IPC 一次完整的抓包流程中概括来说有三个重点工作：同步序列，抓 Beacon 帧确认信道，定信道抓数据帧进行解码。

特点：
IPC 不需要与 APP 建立任何的链路；
数据是单向的，APP 发，IPC 抓，无需交互；
发送端手机 APP 不需要任何“特权” ，只要能发 UDP 广播包即可。

局限性：
单天线设备的手机路由兼容性不好：单天线无法抓到双天线的无线数据，如果 APP 与路由数据交互是基于双天线的，单天线的 IPC 无法抓到无线数据；有的路由不一定会转发 APP 的无线帧，导致空气中只有手机发出的无线帧，单天线不一定能抓到。

二. NVR 概述

(A) 概述

视频监控系统组成

NVR 是 Network Video Recorder 的缩写，即网络录像机。NVR 是通过 IP 网络接入前端音视频采集设备和报警装置，实现监控图像浏览、录像、回放、摄像机控制和报警功能的监控主机设备。

NVR 的基本功能

(B) 重要参数

  • 接入路数

接入路数就是一台 NVR 最大可以管理的 IPC 数量，接入路数越大，说明 NVR 的综合处理能力更强。接入路数是 NVR 选型的决定性因素。
NVR 路数包括：4路、8路、16路、32路、48路、64路、128路、256路或更高。

  • 盘位数量

NVR 在监控网络中的核心作用之一就是存储录像，录像的存储时间和硬盘的总容量有关，硬盘接口越多，接入的硬盘总容量会线性增加，存储时长可以明显提升。
盘位数量是 NVR 选型的重点考虑因素。
当前主流的 NVR 接入盘位数量包括，单盘位、双盘位、四盘位、八盘位以及更高盘位。

  • 解码能力

解码能力，即 NVR 将网络视频流转换为可以输出到显示器画面的能力。
IPC 输出经过编码后的音视频数据流，需要录像机解码后才能输出到显示器上观看。NVR 的解码能力越强，可以同时观看的高清画面数量就越多。解码能力是 NVR 选型的重要考虑因素。
一台 NVR 的解码能力是固定的，例如 800 万像素、3200 万像素等等，为了好理解和记忆，例如 800 万像素的解码能力，经常会表述为 4*1080P/8*720P/16*D1 等。

  • 最大 IPC 接入分辨率

由于 NVR 的实际解码能力有限，根据实际使用和定位，会限制超过自身最大处理能力的 IPC 接入。所以在实际网络中，NVR 的最大接入 IPC 分辨率需要高于所管理的所有 IPC 的最大分辨率。是选型中必须考虑的参数。

  • 编码方式

定义：NVR 对于 H.265 编码方式的支持情况
IPC 主流的编码方式有 H.264 和 H.265，NVR 只有支持 H.265 编码，才可以解码 H.265 的 IPC。仅支持 H.264 编码的 NVR 是无法解码 H.265 编码的视频流。
支持 H.265 编码的 NVR 也支持 H.264。

  • 输出分辨率

输出分辨率是录像机能向外输出画面的分辨率的能力
